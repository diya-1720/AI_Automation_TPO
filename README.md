# SPC Assistant - AI Documentation Platform

![Project Status](https://img.shields.io/badge/Status-Production_Ready-emerald) ![Version](https://img.shields.io/badge/Version-v1.0.0-blue) ![Mobile-Responsive](https://img.shields.io/badge/Mobile-Fully_Responsive-indigo) ![License](https://img.shields.io/badge/License-MIT-green)

An intelligent, AI-powered document generation and report automation platform designed for college **Training & Placement Cells (TPO)**, academic committees, and institutional departments.

SPC Assistant automates the end-to-end event documentation workflow: processing raw event evidence (handwritten notes, scanned papers, images, PDFs, DOCX), performing multi-modal OCR text extraction via Gemini Vision, scoring field confidence, embedding event notice flyers, arranging multi-photo event grids into Word tables, and producing standardized, publication-ready DOCX and PDF reports.

---

## 🌟 Key Features

### 1. 📱 Fully Mobile Responsive Architecture
- **Mobile Navigation Drawer**: Collapsible mobile sidebar with touch-optimized hamburger toggle button for screens under `768px` (320px, 375px, 425px).
- **Adaptive Layout Grids**: Flexbox and CSS Grid structures (`grid-cols-1 md:grid-cols-2`) preventing horizontal scroll across all device width breakpoints (320px–1440px+).
- **Touch-Friendly Controls**: Large tap targets for touchscreens, responsive document previews, and card layouts.

### 2. 👥 Shared Team Account Authentication
- **Team Collaboration Model**: Designed for team collaboration across 20–30 members using single/shared credentials.
- **Global Document Access**: Reports generated by any team member persist globally in `generated/reports_db.json` and are instantly accessible in **Previous Reports**, preventing duplicate report creation.
- **Default Shared Credentials**: Automatically seeds default shared account (`spc@tpo.edu` / `spc12345`) for team access.

### 3. ⚡ AI Auto-Fill & Gemini Vision OCR
- **Handwritten & Printed OCR**: Upload photos of handwritten meeting notes, scanned agendas, or transcripts.
- **Generative Academic LLM**: Uses Gemini Vision & LLM prompts to expand brief inputs into formal academic sections (*Activity Name, Objectives, Methodology, Outcomes, Strengths, Weaknesses, Feedback Summary*).

### 4. 📄 Notice & Brochure Dedicated Upload
- Dedicated file upload area for Notice, Brochure, or Event Announcement flyers.
- Automatically embeds notice images directly into designated sections of output documents with aspect ratio preserved.

### 5. 🖼️ Event Photos Section (Multiple Uploads & 2-Column Grid Layout)
- **Multi-Photo Selection**: Upload multiple event photographs at once (2, 4, 6, 8+ photos) with thumbnail previews and deletion controls.
- **Automated Word Table Grid**: The backend engine arranges all uploaded event photos into a structured 2-column Word table grid on a dedicated page ("Event Photographs"), preserving aspect ratio and preventing chaotic layout overflow.

### 6. ☑️ Consolidated 3-State Checklist Grid
- Replaces individual text inputs with a structured 3-state toggle selector:
  - **✔ Yes** (Checked)
  - **✖ No** (Unchecked)
  - **Blank** (Unset / Optional)

### 7. 📁 Previous Reports & Strict Standardized Naming
- Automatically saves all generated documents into `generated/reports_db.json`.
- Enforces strict dynamic file naming: `[Name_of_the_Activity]_documentation.pdf` / `.docx`.
- Built-in **Previous Reports** viewer with search, preview, and dual export downloads.

### 8. ⚙️ Platform Settings & Global Dark Mode
- **Profile**: Set shared Account Name and associated Email.
- **Appearance**: Toggle switch for **Dark Mode** and **Light Mode** applied globally.
- **Default Auto-Fill Data**: Save default institutional parameters (*Default College Name, Department, Organizer, Venue*) that automatically pre-fill the **New Report** form.
- **Export Preferences**: Configure default download format (*Always download as PDF*, *Always download as DOCX*, or *Ask every time*).

## 🔑 Environment Variables

| Variable Name | Required | Default Value | Description |
|---|---|---|---|
| `PORT` | Yes (Cloud) | `8000` | Port for backend server |
| `GEMINI_API_KEY` | Yes | `""` | Google Gemini API key for Vision OCR & LLM generation |
| `ALLOWED_ORIGINS` | No | `*` | Comma-separated list of allowed CORS origins |
| `API_BASE_URL` | No | `http://localhost:8000` | Base backend API URL |
| `VITE_API_URL` | No | `http://localhost:8000` | Frontend API URL endpoint |

## 🚀 Setup & Local Running

### Prerequisites
- Python 3.10+
- Node.js 18+
- Gemini API Key

### 1. Backend Setup
```bash
cd backend

# Create & activate virtual environment (Windows)
python -m venv venv
.\venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Create .env file
echo GEMINI_API_KEY=your_gemini_api_key_here > .env

# Start FastAPI server
python -m uvicorn main:app --reload --port 8000
```
*(Backend running on `http://localhost:8000`)*

### 2. Frontend Setup
```bash
cd frontend

# Install npm packages
npm install

# Start Vite development server
npm run dev
```
*(Frontend running on `http://localhost:5173`)*

## 🌐 Production Cloud Deployment Guide

The application is structured for instant deployment on cloud platforms (**Render**, **Railway**, **Heroku**, **AWS**, **Vercel**).

### Single Command Production Startup
To start the production server with a single command:
```bash
uvicorn backend.main:app --host 0.0.0.0 --port $PORT
```

### Deploying Backend (Render / Railway / Heroku)
1. Connect your GitHub repository to your cloud provider.
2. Set the **Build Command**:
   ```bash
   pip install -r requirements.txt
   ```
3. Set the **Start Command**:
   ```bash
   uvicorn backend.main:app --host 0.0.0.0 --port $PORT
   ```
4. Configure Environment Variables in Cloud Dashboard:
   - `GEMINI_API_KEY`: Your Google Gemini API Key.
   - `ALLOWED_ORIGINS`: Your frontend domain URL (e.g. `https://spc-assistant.vercel.app`).
   - `API_BASE_URL`: Your backend cloud domain URL (e.g. `https://spc-backend.onrender.com`).

### Deploying Frontend (Vercel / Netlify)
1. Import `frontend/` directory into Vercel/Netlify.
2. Set Environment Variable:
   - `VITE_API_URL`: Your deployed backend API URL (e.g., `https://spc-backend.onrender.com`).
3. Build command: `npm run build`, Output directory: `dist`.

## 🛠️ Troubleshooting

- **PDF Conversion Notice on Linux Cloud Hosts**: On Linux server environments without Microsoft Word installed, `docx2pdf` falls back to serving `.docx` files natively while continuing report generation without breaking API output.
- **CORS Errors**: Ensure `ALLOWED_ORIGINS` in backend environment includes your frontend domain.
- **Missing Gemini API Key**: If `GEMINI_API_KEY` is not set, the platform will utilize intelligent generative fallback defaults so form auto-fill never blocks the user.
